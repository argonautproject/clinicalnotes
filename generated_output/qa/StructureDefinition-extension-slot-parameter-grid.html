NameCard.TypeConstraints and Usage
  Extension 0..1ele-1: All FHIR elements must have a @value or children
ext-1: Must have either extensions or value[x], not both
Definition:
Custom parameters to <code>Slot</code> for additional implementation or site specific context or constraints. For example qualification on visit types such as age restrictions for well child visits.


Comments:
This extension references the <a href="{{site.data.fhir.path}}/parameters.html">Parameters</a> extending its usage outside its stated scope. <code>Parameters</code> has no restful endpoint associated with it and can only be referenced as a <a href="{{site.data.fhir.path}}/references.html#contained">contained</a> resource within <code>Slot</code>.
