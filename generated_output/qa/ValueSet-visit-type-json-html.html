This is intended to facilitates implementation of the IG by provide a common starting point for interoperability.  These concept are used in the Argonaut Appointment, Schedule and Slot resources.  The Visit types can be extended by implementers to meet there use cases.  In order to address overlap between <code>serviceType</code> and <code>appointmentType</code>, these concepts are mapped to the <a href="http://fhir.org/guides/argonaut-scheduling/ConceptMap/visit-appt-map">Argonaut Appointment Types</a>.  A mapping to <a href="http://fhir.org/guides/argonaut-scheduling/ConceptMap/visit-snomed-map">SNOMED CT</a> is also provided.</p>
